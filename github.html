Github

Github er en verktøy for versjonskontroll og temarbeid som bidrar til å strukturere kodeprojekter systematisk.Den har vært en sentral knutepunkt for alle gruppemedlemmer til å samarbeidet enkelt og effektivt. På denne siden skal vi presentere hvordan vi har brukt Github for utføre projketet ved å illustrere prosessen gjennom bilder og forklaring.

1.Versjonskontroll og Kommunikasjon

Github har en kraftig versjonskontrollsystem som gir oss muligheten til å spore og lagrer endringer som en "commit".

Bildet viser et eksempel på hvordan vi samarbeidet og kommuniserte gjennom hele projektet

Et annet eksempel på små endringer vi har gjort underveis

2.Temarbeid

Ved hjelp av "branches" kunne vi jobbe parallet sammen uten å overlappe hverandre.Samtidig opplevde vi utfordringer som merge-konflikt under samarbeidet. Vi brukte github til å gjennømføre merge prosesser mellom branches.

Bildet viser merge-prosessen i Github. Det inkludrer en pull request, merge -commits og en konfliktløsning.Under merge-prosessen kan vi se at det oppsto en konflikt

Dette skjedde fordi to ulike teamedlemmer hadde opptadert samme del av koden samtidig.Vi brukte Githubs merge-verktøy og så at konflikten var knyttet til opptaderinger av menyen.Vi diskuterte endringene og fullførte merge -prosessen med en opptadert struktur

3.Resultat og rapport om bruk av Github

Det hat vært en bratt læringskurve for oss gjennom hele projektet.Vi hadde styr og orden på projektet ved å bruke funkjoner som pull request,issues,branches osv.Til tross for dette, så møtet vi på utfordringer som merge-konflikter og nybegynnerfeil, men dette forsterket våres erfaring i bruk av versjonkontroll og konfliktløsning.Gjennom projketet har vi lært hvordan github støtter utvikling av effektiv samarbeid i webprojekter.
