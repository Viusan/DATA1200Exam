Bruk av Github for effektiv samarbeid

Introduksjon til Git-hub

Github er en verktøy for versjonskontroll og temarbeid som bidrar til å strukturere kodeprojekter systematisk. Den gir oss muligheten til å spore forandring i projektet, og er en sentral knutepunkt for alle gruppemedlemmer der de kan gi tilbakemeldinger, dele koder og dokumentere fremgang i projektet.

Fordeler med GitHub

  • Versjonskontroll; Github har gitt oss en kraftig versjonskontrollsystem gjennom hele projektet. Dette har gitt hos muligheten til å spore og lagre endringer som en "commit" når noen utførte en endring i projektet.
  • Kommunikasjon;Ved hjelp av "commit" kunne vi lett se og forstå hva som ble endret i projektet. Vi opptaderte hverandre om projektet ved å legge til en en kort besrkivelser om hva som ble endret og hvem som utførte endringene.
  • Temarbeid;GitHub har gjort det mulig for oss å arbeidet med projektet samtidig uten å overlappe elller kollidere med hverandre. Vi kunne jobbe parallet sammenn ved hjelp "branches" noe som forsterket våres samarbeidsevner og kvalitet.
